Added `backend.explain(expression, options)` interface with typed, backend-specific result objects. SQLite backends return `SQLiteExplainResult` (bytecode) or `SQLiteExplainQueryPlanResult` (query plan), both with built-in index-usage analysis helpers (`analyze_index_usage()`, `is_full_scan`, `is_index_used`, `is_covering_index`). The interface follows the sync/async parity principle via `SyncExplainBackendMixin` and `AsyncExplainBackendMixin`.
